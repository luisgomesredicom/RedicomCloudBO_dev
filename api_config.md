# Documentação de Configuração da API (`api_config.php`)

## Visão Geral

Este documento detalha as configurações presentes no ficheiro `api_config.php`, que são cruciais para o funcionamento da plataforma. Estas configurações abrangem desde o comportamento da aplicação (APP), gestão de cache, otimização de imagens, até parâmetros específicos de templates HTML. A manipulação incorreta destas variáveis pode ter um impacto significativo na segurança, desempenho e funcionalidade do frontend e backend.

**Importante:** Este ficheiro contém configurações sensíveis. Qualquer alteração deve ser feita com extrema cautela e após validação rigorosa.

## Regras para o Agente IA

Ao interagir com as configurações definidas em `api_config.php`, o agente IA deve aderir às seguintes diretrizes:

1.  **Análise Abrangente**: Antes de propor qualquer alteração ou de inferir um comportamento, o agente deve analisar todas as configurações relevantes e as suas interdependências.
2.  **Validação de Impacto**: Para cada configuração, o agente deve considerar o seu impacto potencial na segurança, desempenho, UX e integração com outros módulos.
3.  **Prioridade à Segurança**: Configurações relacionadas com chaves de API, IDs de aplicações e endpoints devem ser tratadas com a máxima prioridade de segurança. Nunca devem ser expostas ou alteradas sem autorização explícita e validação.
4.  **Não Assumir Valores Padrão**: Embora alguns valores padrão sejam fornecidos, o agente não deve assumir que estes são os valores em produção. Deve sempre referenciar o valor atual no ficheiro.
5.  **Questionamento Proativo**: Em caso de incerteza sobre a finalidade ou o impacto de uma configuração, o agente deve procurar esclarecimentos antes de prosseguir.
